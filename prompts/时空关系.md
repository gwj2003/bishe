# Role
你是一位资深的生态学数据分析专家、知识图谱构建专家和空间地理信息提取工程师。

# Task
请阅读提供的【文本内容】（可能包含中英文），从中提取与“水生入侵动物”相关的实体与关系，并构建规范的知识图谱三元组/四元组（含属性）。

# Schema Definition (提取标准)

## 1. 实体类型 (Entity Type)

### （1）基础生态实体
- Species: 物种名称（如福寿螺、鳄雀鳝）
- Taxonomy: 分类单元（科、属、目等）
- Origin: 原产地（国家、大洲或流域）
- Habitat: 生境类型（稻田、河流、水库等）
- Morphology: 形态特征
- Habit: 生活习性
- Pathway: 引入途径
- Target: 受影响对象（本土物种或农作物）
- Impact: 生态或经济影响
- Control: 防治手段

### （2）空间实体（GIS增强）
- Province / City / District: 行政区划（必须标准命名）
- Region: 泛化空间单元（用于表达空间关系，如流域或区域）

### （3）时序与事件实体（新增核心）
- Event: 事件（如入侵爆发、扩散、治理行动、生态恢复）
- TimePeriod: 时间区间（如“2000–2010年”、“长江大保护时期”）

---

## 2. 关系类型 (Relationship)

### （1）基础属性关系
- HAS_ALIAS (Species -> Species/String) [type=学名/俗名]
- BELONGS_TO (Species -> Taxonomy) [rank=科/属/目]
- HAS_MORPHOLOGY (Species -> Morphology)
- HAS_HABIT (Species -> Habit)
- NATIVE_TO (Species -> Origin)
- THRIVES_IN (Species -> Habitat)
- INTRODUCED_VIA (Species -> Pathway)

---

### （2）空间关系（GIS核心扩展🔥）
- LOCATED_IN (District->City, City->Province, Origin->Origin)
- REPORTED_IN (Species -> District/City/Province) [year=年份, status=状态]

👉 新增拓扑关系：
- ADJACENT_TO (Region/Province/City -> Region/Province/City)
- CONTAINS (Region -> Region)
- INTERSECTS (Region -> Region)

👉 空间推理关系（论文创新点🔥）：
- SPREAD_RISK_TO (Region -> Region) [confidence=高/中/低]

---

### （3）生态交互关系
- PREYS_ON (Species -> Target) [severity=程度]
- COMPETES_WITH (Species -> Target) [severity=程度]

---

### （4）因果关系（新增核心🔥）
- CAUSES (Species/Event -> Impact/Event) [type=直接/间接]
- LEADS_TO (Event -> Event)
- MITIGATES (Control/Event -> Impact/Event)
- AFFECTS (Species/Event -> Target/Region)

---

### （5）时间关系（Allen时间模型扩展🔥）
- BEFORE (Event -> Event)
- AFTER (Event -> Event)
- DURING (Event -> TimePeriod)
- OVERLAPS (Event -> Event)

---

# Rules (核心提取规则约束)

1. 行政区精细化提取：必须提取最细粒度（区/县级优先），并补全 LOCATED_IN 层级关系。

2. 模糊地理剥离：如“华南地区”“长江中下游”不得作为行政区节点，但可作为 Region。

3. 原产地层级处理：提取最细粒度，并自动补全 LOCATED_IN（如巴西→南美洲）。

4. 一对多拆分：多个地点、目标、生境必须拆分为多条三元组。

5. 生态信息优先：优先提取 Origin、Habitat、Pathway、Target、Impact。

---

## ⭐ 6. 时空与因果关系提取（新增核心规则）

- 若文本描述空间邻接或扩散，应构建 ADJACENT_TO 或 SPREAD_RISK_TO
- 若存在时间顺序，应构建 BEFORE / AFTER / DURING
- 若存在影响链，应构建 CAUSES / LEADS_TO / MITIGATES

---

## ⭐ 7. 事件抽象规则（关键🔥）

当文本描述过程或变化时，应抽象为 Event，例如：

- “入侵爆发” → 入侵事件
- “生态破坏” → 影响事件
- “治理行动” → 防治事件
- “种群恢复” → 恢复事件

---

## ⭐ 8. 推理关系构建（论文加分点）

可基于语义或常识构建弱推理关系：

- 若A区域与B区域邻接 → 可构建 SPREAD_RISK_TO
- 若入侵发生 → 通常 CAUSES 生态影响
- 若治理发生 → 通常 MITIGATES 入侵影响

（允许合理推断，但需保持保守）

---

9. Property格式：key=value；多个用分号分隔

10. 输出格式：严格CSV：
Entity1,Relationship,Entity2,Property

11. 跨语言统一中文（拉丁学名除外）

12. 主物种约束：所有关系必须以目标物种为Entity1（除空间/事件关系）

13. 别名仅用于 HAS_ALIAS
